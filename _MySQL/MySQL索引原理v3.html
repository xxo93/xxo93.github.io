B+ 树结构
B+ 树是 InnoDB 的默认索引结构。所有数据存储在叶子节点，叶子节点通过链表相连，支持高效范围查询。拖动图表可查看被遮挡的节点。
🌳

多路平衡

每个节点可有多个子节点，树高通常 3~4 层即可存储千万级记录。

📄

数据在叶子节点

内部节点只存索引键和指针，所有实际数据都在叶子节点中。

🔗

叶子节点链表

叶子节点通过指针相连，支持高效的范围查询（BETWEEN、ORDER BY）。

⚡

查找效率 O(log n)

1500 万条记录只需 3~4 次磁盘 I/O 即可定位。

B+ 树 vs B 树
B+ 树（MySQL 使用）
数据只在叶子节点 → 内部节点更小 → 每页存更多索引键 → 树更矮 → I/O 更少。叶子链表支持范围扫描。
B 树
数据在所有节点 → 内部节点更大 → 树更高 → 更多 I/O。不支持高效范围查询。
聚簇索引（Clustered Index）
聚簇索引将数据行直接存储在 B+ 树的叶子节点中。找到主键就直接拿到了完整行数据，无需额外查找。拖动图表查看完整树结构。

覆盖索引（Covering Index）
当查询所需的所有字段都包含在索引中时，MySQL 无需回表，直接从索引返回结果。
✗ 需要回表 · SELECT *

② 找到 name='Ivy' → id=9
③ ⚠️ 回表：用 id=9 回聚簇索引

最左前缀原则速查
索引 (a, b, c)
✓ WHERE a=1 · ✓ WHERE a=1 AND b=2 · ✓ WHERE a=1 AND b=2 AND c=3
⚠ WHERE a=1 AND c=3（只用到 a）
✗ WHERE b=2 · ✗ WHERE c=3 · ✗ WHERE b=2 AND c=3
