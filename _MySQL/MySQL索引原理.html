💡 关键概念

聚簇索引（Clustered Index）

InnoDB 的主键索引是一棵 B+ 树，叶子节点存储完整的行数据。整张表的数据按主键顺序存储在叶子节点中，所以聚簇索引 = 数据本身。通过主键查询时，找到叶子节点就拿到了全部数据。

二级索引 & 回表

非主键索引（如 idx_age）的叶子节点只存索引列的值 + 主键值。当 SELECT 需要其他列时，必须用主键值再回聚簇索引查完整数据，这个过程叫回表，会产生额外的 IO 开销。

覆盖索引（Covering Index）

如果查询所需的所有列都包含在索引中（索引列 + 主键），就不需要回表，直接从二级索引返回数据。这是重要的 SQL 优化手段，可以通过合理建立联合索引来实现。
