B+ 树结构
B+ 树是 InnoDB 的默认索引结构。所有数据在叶子节点，叶子间通过链表相连。修改上方数据表，此处自动更新。
🌳

多路平衡

每个节点最多 3 个键（演示用，实际 MySQL 为数百），3~4 层即可存储千万级记录。

📄

数据在叶子节点

内部节点只存索引键和指针，所有实际数据都在叶子节点中。

🔗

叶子节点链表

叶子节点通过指针相连，支持高效的范围查询。

⚡

查找效率 O(log n)

1500 万条记录只需 3~4 次磁盘 I/O 即可定位。

索引 (a, b, c)
✓ WHERE a = 1
✓ WHERE a = 1 AND b = 2
✓ WHERE a = 1 AND b = 2 AND c = 3
⚠ WHERE a = 1 AND c = 3（只用 a）
✗ WHERE b = 2（跳过 a）
✗ WHERE c = 3（跳过 a, b）
