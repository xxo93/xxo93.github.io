LeetCode 1343 · 滑动窗口 · Easy

大小为 K 且平均值大于等于阈值的子数组数目

给定一个整数数组 arr 和两个整数 k 与 threshold，请你找出长度恰好为 k 且平均值 ≥ threshold 的连续子数组有多少个。

📋题意解读

简单来说，这道题在问什么？

想象你有一排数字：

[ 2, 2, 2, 2, 5, 5, 5, 8 ]

题目要求你拿一个长度固定为 k 的"框"，从左到右依次框住连续的 k 个数，算出它们的平均值。

如果平均值 ≥ threshold（阈值/门槛），就算"达标"一次。

最后问你：一共达标了几次？

生活类比

就像你拿一把固定长度的尺子，沿着一排刻度从左滑到右：

  • 尺子每滑一格，就量一次盖住的那些数的平均值
  • 平均值够高（≥ threshold）就记一笔
  • 滑到最后，看看总共记了几笔

示例推演

arr = [2,2,2,2,5,5,5,8]，k = 3，threshold = 4

  • [2, 2, 2] → 平均值 2.00 → 不达标 ✗
  • [2, 2, 2] → 平均值 2.00 → 不达标 ✗
  • [2, 2, 5] → 平均值 3.00 → 不达标 ✗
  • [2, 5, 5] → 平均值 4.00 → 达标 ✓
  • [5, 5, 5] → 平均值 5.00 → 达标 ✓
  • [5, 5, 8] → 平均值 6.00 → 达标 ✓

答案：3（有 3 个子数组达标）

🔍为什么用滑动窗口？

暴力做法的问题

最直觉的方法：每次选 k 个数，重新加一遍算总和。能做对，但重复计算太多。

比如 [2, 2, 2, 5]，k=3：

  • 第一次算 [2, 2, 2]，把 2+2+2 加了一遍
  • 第二次算 [2, 2, 5]，又要重新加 2+2+5

两个窗口只有一头一尾不同（左边的 2 出去了，右边的 5 进来了），中间的 [2, 2] 完全一样，没必要重复加。

滑动窗口的巧妙之处

想象你在算移动平均：

  • 减掉左边离开窗口的那个数
  • 加上右边新进入窗口的那个数

一步操作就能从旧窗口的总和得到新窗口的总和，不需要重新遍历 k 个数！

核心思想：维护一个"窗口"，每次只处理变化的部分。

📐算法具体步骤

  1. 计算前 k 个元素的总和 window_sum，作为初始窗口的值。
  2. 把达标条件转换为：window_sum ≥ k × threshold（比较整数比比较小数更高效）。
  3. 检查第一个窗口是否达标，是则 count += 1。
  4. 开始滑动：窗口从位置 0 滑到位置 n − k，每次滑动时：
    • 减去左边离开窗口的元素 arr[i − k]
    • 加上右边新进入窗口的元素 arr[i]
  5. 每滑一步，检查新窗口总和是否达标，是就 count += 1。
  6. 全部滑完后返回 count。

💻代码实现（Python）

solution.py 
class Solution:
    def numOfSubarrays(self, arr: List[int], k: int, threshold: int) -> int:
        # 达标要求的最小总和
        target = k * threshold

        # 第一步：计算第一个窗口的总和
        window_sum = sum(arr[:k])

        # 第二步：检查第一个窗口是否达标
        count = 1 if window_sum >= target else 0

        # 第三步：滑动窗口
        for i in range(k, len(arr)):
            # 减去左边离开的，加上右边新来的
            window_sum += arr[i] - arr[i - k]
            # 检查是否达标
            if window_sum >= target:
                count += 1

        return count

📊复杂度分析

时间复杂度

O(n)

只遍历了一遍数组。计算初始窗口 O(k)，之后每次滑动 O(1)，共滑 n−k 次，总计 O(n)。

空间复杂度

O(1)

只用了几个变量（window_sum、count、target），不需要额外数组或数据结构。

与暴力法对比

方法时间复杂度空间复杂度
暴力法（每次重新求和） O(n × k) O(1)
滑动窗口 O(n) O(1)

当 k 较大时优势非常明显。比如 n=100000, k=50000，暴力法约 50 亿次加法，滑动窗口只需约 10 万次。
