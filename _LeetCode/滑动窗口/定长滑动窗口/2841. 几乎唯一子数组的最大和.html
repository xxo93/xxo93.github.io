几乎唯一子数组的最大和

给定一个整数数组 nums 和两个整数 m 与 k，如果一个长度为 k 的子数组包含至少 m 个不同元素，就称它是"几乎唯一"的。请找出所有几乎唯一子数组中的最大和，不存在则返回 0。

📋题意解读

简单来说，这道题在问什么？

给你一排数字：

[ 2, 6, 7, 3, 1, 7 ]

用一个长度固定为 k 的"框"从左到右滑动，每次框住 k 个连续数字。

但这次不是算平均值，而是要同时满足两个条件：

  • 框里的数字至少有 m 种（不同元素的个数 ≥ m）
  • 在满足条件的框中，找出元素之和最大的那个

如果框里有重复数字，只算一种。比如 [7, 3, 1, 7] 中 7 出现了两次，但不同元素只有 3 种（7、3、1）。

生活类比

想象你在水果摊前挑水果篮：

  • 每个篮子必须装 k 个水果
  • 篮子里至少要有 m 种不同的水果（不能全是苹果）
  • 在所有达标的篮子里，选总价最高的那个

示例推演

nums = [2, 6, 7, 3, 1, 7]，m = 3，k = 4

  • [2, 6, 7, 3] → 4 种 ≥ 3 ✓ → 和 = 18
  • [6, 7, 3, 1] → 4 种 ≥ 3 ✓ → 和 = 17
  • [7, 3, 1, 7] → 3 种 ≥ 3 ✓ → 和 = 18

答案：18（最大和为 18）

🔍为什么用滑动窗口？

暴力做法的问题

最直觉的方法：枚举每个长度为 k 的子数组，用 set 去重算不同元素个数，再求和。每次操作 O(k)，总共 O(n × k)。

问题在于：相邻两个窗口只差一头一尾，中间的元素完全一样，却要重复遍历。

滑动窗口 + 频率表的巧妙之处

维护一个频率表（字典），记录窗口内每个数字出现了几次：

  • 移除左边离开的数字：频率 −1，如果减到 0 就从表里删掉（不同元素数 −1）
  • 添加右边新进的数字：如果之前不在表里（不同元素数 +1），频率 +1
  • 同时维护窗口总和：减去离开的、加上新来的

每一步只需 O(1) 操作，就能同时更新不同元素数和窗口总和！

核心思想：用频率表增量维护不同元素数，避免重复遍历。

📐算法具体步骤

  1. 初始化：用前 k 个元素构建频率表 freq，计算不同元素数 distinct 和窗口总和 win_sum。
  2. 检查第一个窗口：如果 distinct ≥ m，记录 max_sum = win_sum。
  3. 开始滑动窗口（左指针从 0 到 n−k）：
    • 移除左边元素 nums[left]：频率 −1，若减到 0 则 distinct − 1 并从表中删除
    • 添加右边元素 nums[left + k]：若不在表中则 distinct + 1，频率 +1
    • 更新 win_sum：减去离开的、加上新来的
  4. 每滑一步，如果 distinct ≥ m，用 win_sum 更新 max_sum。
  5. 全部滑完后返回 max_sum（若从未达标则返回 0）。

💻代码实现（Python）

solution.py 
from collections import defaultdict

class Solution:
    def maxSum(self, nums: List[int], m: int, k: int) -> int:
        freq = defaultdict(int)
        distinct = 0
        win_sum = 0
        max_sum = 0
        n = len(nums)

        # 第一步：构建初始窗口
        for i in range(k):
            v = nums[i]
            if freq[v] == 0:
                distinct += 1
            freq[v] += 1
            win_sum += v

        # 第二步：检查第一个窗口
        if distinct >= m:
            max_sum = win_sum

        # 第三步：滑动窗口
        for i in range(k, n):
            # 移除左边元素
            leave = nums[i - k]
            freq[leave] -= 1
            if freq[leave] == 0:
                distinct -= 1
            win_sum -= leave

            # 添加右边元素
            enter = nums[i]
            if freq[enter] == 0:
                distinct += 1
            freq[enter] += 1
            win_sum += enter

            # 更新最大和
            if distinct >= m:
                max_sum = max(max_sum, win_sum)

        return max_sum

📊复杂度分析

时间复杂度

O(n)

构建初始窗口 O(k)，之后每次滑动 O(1)，共滑 n−k 次，总计 O(n)。

空间复杂度

O(k)

频率表最多存储 k 个不同元素（窗口大小上限），因此空间为 O(k)。

与暴力法对比

方法时间复杂度空间复杂度
暴力法（每次用 set 去重 + 求和） O(n × k) O(k)
滑动窗口 + 频率表 O(n) O(k)

当 n 和 k 都较大时优势非常明显。例如 n=10⁵, k=5×10⁴，暴力法约 50 亿次操作，滑动窗口只需约 10 万次。
