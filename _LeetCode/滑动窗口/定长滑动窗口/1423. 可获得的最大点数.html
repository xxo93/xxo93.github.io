LeetCode 1423 · 滑动窗口 · Medium

可获得的最大点数

给定整数数组 cardPoints 和整数 k，每次可以从数组的最左端或最右端取一张牌，取恰好 k 张，返回可获得的最大点数。

通俗讲解

想象你面前有一排牌，每张牌上有个数字代表分数。规则很简单：

  • 你只能从最左边或最右边抽牌（不能从中间抽）
  • 必须正好抽 k 张
  • 目标：让抽到的牌的数字之和最大

以 [1, 2, 3, 4, 5, 6, 1]，k=3 为例，所有可能的取法：

牌组:  [1] [2] [3] [4] [5] [6] [1]    总和 = 22, k = 3

方案1: 左取0张，右取3张          [5] [6] [1]  = 12  ✓ 最大
方案2: 左取1张，右取2张  [1]          [6] [1]  =  8
方案3: 左取2张，右取1张  [1] [2]      [1]     =  4
方案4: 左取3张，右取0张  [1] [2] [3]          =  6

关键洞察：换个角度想——你不是在选"要抽哪些牌"，而是在选"不抽哪些牌"。不抽的牌一定是一段连续的中间部分（大小为 n-k）。所以问题变成：找到一段连续的 n-k 张牌，让它们的和最小。

公式：答案 = 总和 - 最小窗口和，用滑动窗口即可 O(n) 求解。

算法解析

为什么用滑动窗口？

暴力做法：枚举从左边取 0, 1, 2, ..., k 张，对应右边取 k, k-1, ..., 0 张，一共 k+1 种方案，逐个求和比较。时间 O(k)。

但我们发现一个规律：相邻两种方案之间只差一张牌——左边多取一张、右边少取一张。这正是滑动窗口的特征！

换一个更优雅的视角：总共取 k 张，意味着不取中间连续的 n-k 张。问题转化为：在数组中找一段长度为 n-k 的连续子数组，使其和最小。答案 = 总和 - 最小窗口和。

具体步骤

  • 计算数组总和 total。
  • 计算初始窗口（前 n-k 个元素）的和 windowSum。
  • 窗口每次右移一格：减去左端离开的元素，加上右端新进入的元素。
  • 每滑动一次，用 min(minWindowSum, windowSum) 跟踪最小窗口和。
  • 遍历结束，返回 total - minWindowSum。

代码（Python）

class Solution:
    def maxScore(self, cardPoints: List[int], k: int) -> int:
        n = len(cardPoints)
        win_size = n - k  # 不选取的窗口大小
        total = sum(cardPoints)

        # 特殊情况：k == n，全部选取
        if win_size == 0:
            return total

        # 初始化第一个窗口的和
        window_sum = sum(cardPoints[:win_size])
        min_window_sum = window_sum

        # 滑动窗口：每次右移一格
        for i in range(1, k + 1):
            # 左端离开: cardPoints[i-1]
            # 右端进入: cardPoints[i+win_size-1]
            window_sum += cardPoints[i + win_size - 1] - cardPoints[i - 1]
            min_window_sum = min(min_window_sum, window_sum)

        return total - min_window_sum

复杂度分析

时间复杂度：O(n) — 先求总和 O(n)，再滑动窗口遍历 k 次 O(k)，总计 O(n+k) = O(n)。

空间复杂度：O(1) — 只用了 total、windowSum、minWindowSum 等常数个变量。
