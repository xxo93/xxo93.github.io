算法解析

题目理解

想象一个仓库，每天都有新货品运来，货品有不同编号（类型）。我们有一个滚动窗口，覆盖最近 w 天。规定：这个窗口内，同一种类型的货品最多只能保留 m 个。如果某天运来的货品会导致某种类型在窗口内超过 m 个，就必须丢弃这一天的货品。

目标是用最少的丢弃来满足这个条件。

为什么用滑动窗口？

暴力做法：枚举所有可能的丢弃组合，检查是否满足条件，时间指数级。

关键观察：当窗口从位置 i 移到 i+1 时，只有一个旧元素离开（位置 i-w+1）和一个新元素进入（位置 i+1）。频次表只需做"一减一加"，不用重新统计。

贪心策略：从左到右遍历，对于位置 i，如果保留 arrivals[i] 不会导致当前窗口中该类型频次超过 m，就保留；否则丢弃。这个贪心是正确的，因为越靠右的元素，它所在的有效窗口越多（受约束更多），所以优先保留它能最大化保留数量。

具体步骤

  1. 用一个队列（双端队列）维护当前窗口内被保留的元素 (位置, 类型)。
  2. 用一个哈希表统计窗口内各类型的频次。
  3. 遍历每个位置 i：
    • 如果队首元素的位置 < i - w + 1（已滑出窗口），则出队并减少该类型频次。
    • 检查 arrivals[i] 当前在窗口中的频次是否已达 m。
    • 如果已达上限 → 丢弃（丢弃数 +1，不入队）。
    • 否则 → 保留（入队，频次 +1）。
  4. 遍历结束，累计丢弃数即为答案。

代码（Python）

from collections import deque

class Solution:
    def minDiscards(self, arrivals: list[int], w: int, m: int) -> int:
        q = deque()          # 存储 (位置, 类型) 的队列
        freq = {}            # 窗口内各类型频次
        discards = 0

        for i in range(len(arrivals)):
            # 1. 滑出窗口：移除超出范围的元素
            while q and q[0][0] < i - w + 1:
                _, val = q.popleft()
                freq[val] -= 1

            # 2. 检查当前类型在窗口中是否已达上限
            v = arrivals[i]
            if freq.get(v, 0) >= m:
                discards += 1   # 超限，丢弃
            else:
                q.append((i, v)) # 保留，入队
                freq[v] = freq.get(v, 0) + 1

        return discards

复杂度分析

时间复杂度：O(n) — 每个元素最多入队一次、出队一次，总操作为 2n。

空间复杂度：O(w) — 队列最多存储 w 个元素，哈希表最多 w 个键。
