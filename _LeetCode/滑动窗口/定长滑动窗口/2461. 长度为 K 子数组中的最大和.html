算法解析

为什么用滑动窗口？

暴力做法：枚举所有长度为 k 的子数组，逐个检查元素是否全部不同，再求和取最大值。时间复杂度 O(n·k)，当 n 很大时会超时。

关键观察：相邻两个窗口只差一进一出。用一个哈希表记录窗口内每个元素的出现次数，右指针往右扩展时加入新元素，遇到重复就收缩左指针直到重复消失，窗口超过 k 也收缩。每个元素最多进出窗口各一次，O(n) 搞定。

具体步骤

  • 初始化左指针 left = 0，当前和 sum = 0，最大和 maxSum = 0，频率表 freq = {}
  • 右指针 right 从 0 遍历到 n−1：
    • 将 nums[right] 加入窗口（sum 加上它，freq 对应计数 +1）
    • 如果 freq[nums[right]] > 1（出现重复），不断移除 nums[left] 并 left++，直到不重复
    • 如果窗口大小 > k，不断移除 nums[left] 并 left++，直到窗口大小 ≤ k
    • 如果窗口大小恰好等于 k（此时必然所有元素互不相同），更新 maxSum = max(maxSum, sum)
  • 遍历结束，返回 maxSum

为什么窗口大小 = k 时一定没有重复？

因为 while 循环的终止条件是「freq[nums[right]] ≤ 1 且窗口大小 ≤ k」。当窗口大小恰好为 k 时，while 循环已经保证了没有任何元素出现次数 > 1，所以所有元素必然互不相同。

代码

class Solution:
    def maximumSubarraySum(self, nums: List[int], k: int) -> int:
        freq = {}
        left = 0
        current_sum = 0
        max_sum = 0

        for right in range(len(nums)):
            # 将 nums[right] 加入窗口
            current_sum += nums[right]
            freq[nums[right]] = freq.get(nums[right], 0) + 1

            # 有重复 或 窗口过大 → 收缩左边界
            while freq[nums[right]] > 1 or (right - left + 1) > k:
                freq[nums[left]] -= 1
                if freq[nums[left]] == 0:
                    del freq[nums[left]]
                current_sum -= nums[left]
                left += 1

            # 窗口大小恰好为 k → 更新答案
            if right - left + 1 == k:
                max_sum = max(max_sum, current_sum)

        return max_sum

复杂度分析

时间复杂度：O(n) — 每个元素最多被加入窗口一次、移出窗口一次，总共 2n 次操作。

空间复杂度：O(k) — 哈希表最多存储窗口内的 k 个不同元素。
