半径为 k 的子数组平均值

给定一个 0-indexed 的整数数组 nums 和整数 k，返回一个数组 avgs，其中 avgs[i] 是以索引 i 为中心、半径为 k 的子数组的平均值（整数除法）。如果某一侧元素不够 k 个，则 avgs[i] = -1。

📋题意解读

简单来说，这道题在问什么？

想象你站在一排数字的某个位置上，你的视野半径是 k——也就是说你能看到左边 k 个数和右边 k 个数（加上你自己，一共 2k+1 个数）。

题目要求你从最左边走到最右边，每到一个位置就算一次视野内所有数的平均值（向下取整）。

但是！如果你站得太靠边——左边或右边凑不齐 k 个数——那就没法算，结果记为 -1。

生活类比

就像你拿着一个固定宽度的取景框拍照：

  • 取景框宽度 = 2k + 1，你在正中间
  • 站在人群中间时，取景框刚好框住 2k+1 个人，算出他们的平均身高
  • 站在太靠左或太靠右时，取景框的一边会超出人群范围——没法拍，结果为 -1

示例推演

nums = [7, 4, 3, 9, 1, 8, 5, 2, 6]，k = 3，窗口大小 = 2×3+1 = 7

  • 位置 0：左边缺 3 个 → -1
  • 位置 1：左边缺 2 个 → -1
  • 位置 2：左边缺 1 个 → -1
  • 位置 3：[7,4,3,9,1,8,5] → 总和 37 → 平均 5
  • 位置 4：[4,3,9,1,8,5,2] → 总和 32 → 平均 4
  • 位置 5：[3,9,1,8,5,2,6] → 总和 34 → 平均 4
  • 位置 6：右边缺 1 个 → -1
  • 位置 7：右边缺 2 个 → -1
  • 位置 8：右边缺 3 个 → -1

答案：[-1, -1, -1, 5, 4, 4, -1, -1, -1]

🔍为什么用滑动窗口？

暴力做法的问题

最直觉的方法：对每个有效位置，重新把 2k+1 个数加一遍。能做对，但重复计算太多。

比如上面的例子，位置 3 的窗口是 [7,4,3,9,1,8,5]，位置 4 的窗口是 [4,3,9,1,8,5,2]。

两个窗口有 [4,3,9,1,8,5] 这 6 个数完全一样，只有"一头一尾"不同——左边的 7 出去了，右边的 2 进来了。没必要重新加一遍！

滑动窗口的巧妙之处

窗口每向右滑一格，你只需要做两步操作：

  • 减去最左边离开窗口的那个数
  • 加上最右边新进入窗口的那个数

一步操作就能从旧窗口的总和得到新窗口的总和，O(1) 完成！

核心思想：维护一个"窗口总和"，每次只处理变化的部分。

📐算法具体步骤

  1. 计算窗口大小 w = 2k + 1。如果 w > n（数组长度），说明没有任何位置能凑齐足够的元素，直接返回全 -1 的数组。
  2. 计算前 w 个元素的总和 window_sum，作为第一个有效窗口（中心在索引 k）的值。
  3. 将 result[k] = window_sum // w（整数除法 = 向下取整）。
  4. 开始滑动窗口：中心从 k+1 到 n-1-k，每次滑动时：
    • 减去左边离开窗口的元素 nums[i - k - 1]
    • 加上右边新进入窗口的元素 nums[i + k]
  5. 每滑一步，计算 result[i] = window_sum // w。
  6. 对于索引 0 ~ k-1 和 n-k ~ n-1 的位置，结果直接设为 -1。
  7. 返回结果数组 result。

💻代码实现（Python）

solution.py 
class Solution:
    def getAverages(self, nums: List[int], k: int) -> List[int]:
        n = len(nums)
        w = 2 * k + 1          # 窗口大小
        result = [-1] * n      # 默认全 -1

        # 窗口比数组还大 → 全是 -1
        if w > n:
            return result

        # 第一步：计算第一个窗口的总和
        window_sum = sum(nums[:w])
        result[k] = window_sum // w

        # 第二步：滑动窗口
        for i in range(k + 1, n - k):
            # 减去离开的，加上新来的
            window_sum += nums[i + k] - nums[i - k - 1]
            result[i] = window_sum // w

        return result

📊复杂度分析

时间复杂度

O(n)

计算第一个窗口 O(w)，之后每次滑动 O(1)，共滑 n−w 次。总计 O(n)。

空间复杂度

O(1)

除了返回的结果数组外，只用了 window_sum 等常数个变量。额外空间 O(1)。

与暴力法对比

方法时间复杂度空间复杂度
暴力法（每个位置重新求和） O(n × k) O(1)
滑动窗口 O(n) O(1)

当 k 较大时优势非常明显。比如 n=100000, k=50000，暴力法约 50 亿次加法，滑动窗口只需约 10 万次。
